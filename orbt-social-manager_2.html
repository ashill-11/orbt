Welcome to ORBT
ORBT syncs with Supabase so your whole team shares the same data in real time. Setup takes about 3 minutes.
1
Create a free Supabase project
Go to supabase.com → New project → choose a name and password → wait ~1 min for it to start.
2
Run this SQL to create tables
In your project → SQL Editor → paste and run this:

create table accounts (id text primary key, data jsonb); create table posts (id text primary key, data jsonb); create table team (id text primary key, data jsonb); create table notifications (id text primary key, data jsonb); alter table accounts enable row level security; alter table posts enable row level security; alter table team enable row level security; alter table notifications enable row level security; create policy "public" on accounts for all using (true); create policy "public" on posts for all using (true); create policy "public" on team for all using (true); create policy "public" on notifications for all using (true);
3
Paste your project URL and anon key
Settings → API → copy Project URL and anon public key below.

Make.com — Real Publishing Setup
ORBT sends your posts to Make.com via webhook. Make.com then publishes to the real platform (Twitter/X, LinkedIn, etc.) on your behalf. Make.com is free up to 1000 operations/month — no credit card needed.
Twitter / X
1. Go to make.com → Create new scenario → Add Webhooks → Custom webhook trigger → Copy the URL
2. Add Twitter → Create a Tweet action → map {{1.content}} to the tweet text field → Activate
3. Paste the webhook URL in your account settings (Add account → Webhook URL)
LinkedIn
Same flow — use LinkedIn → Create a post module. Map {{1.content}} to the text field.
Instagram
Instagram requires a Business account connected to a Facebook Page. Use Instagram for Business → Create a photo post module. You can pass an image URL via {{1.image_url}} if you store images somewhere publicly accessible.
What ORBT sends in the webhook payload:

{ "content": "your post text", "account": "@handle", "platform": "x", "scheduled_date": "2024-01-30", "scheduled_time": "16:30" }
